# Netly Ledger — 用户手册与参考指南

欢迎使用 **Netly Ledger**，这是一个本地优先、注重隐私的个人财务引擎。Netly Ledger 100% 在您的设备上运行：所有账户、交易和规则都存储在本地 SQLite 数据库中。没有云端服务器，没有广告，也没有跟踪您财务生活的遥测技术。

本手册详细介绍了 Netly Ledger 中的每个功能、工作流和设置。

## 1. 入门与安全

### 加密安全 (PassKeys)
Netly Ledger 使用 **WebAuthn (PassKeys)** 来保护对本地账簿的访问。PassKeys 用与您的物理设备（如 Touch ID、Face ID 或 Windows Hello）绑定的加密安全凭据代替了传统的密码。
- **首次设置**：首次启动应用程序时，系统会提示您输入**设备名称**（例如“我的 MacBook Air”）并注册您的第一个 PassKey。
- **后续登录**：只需点击“使用 PassKey 登录”并在您的设备上进行身份验证。
- **多设备同步**：如果您想从另一台不共享凭据同步网络的设备（例如手机或第二台电脑）访问此应用：
  1. 在已注册的设备上，前往**设置** -> **PassKeys**。
  2. 点击**添加设备** / **生成设置验证码**。这将创建一个单次使用的设置验证码，有效期为 15 分钟。
  3. 在新设备上，导航到登录页面，选择**使用设置验证码**，然后输入该验证码以引导您的新凭据。

### 数据隐私与存储
所有数据都保存在安装目录中位于 `prisma/dev.db` 的单文件 SQLite 数据库中。会话签名密钥在首次运行时自动生成，并本地存储在 `.session-secret` 中。该文件的备份和维护完全由您控制。

## 2. 仪表板概览

**仪表板**是 Netly Ledger 的财务驾驶舱。它汇总了您所有账户的账簿数据，以提供您财务状况的实时摘要。

### 关键财务指标
在仪表板顶部有四个关键指标卡片：
1. **净资产**：所有资产减去所有负债后的合并余额。它以您选择的货币计算。
2. **净收入**：所选期间的总收入减去总支出，并附有与前一期间相比的百分比变化。
3. **储蓄率**：您储蓄的收入百分比（净收入 / 总收入）。建议目标为 **20%+**。
4. **资金跑道**：根据您的滚动平均月度支出（消耗率），估算您的流动资产可以维持多少个月。如果您的净现金流为正，卡片将显示**现金流为正**。

### 互动图表
- **净资产趋势**：折线图，显示您在所选期间的净资产变化。如果您拥有多种货币的账户，可以按货币筛选图表。
- **收入与支出**：对比柱状图，显示总收入与总支出。
- **现金流指标**：提供按**经营活动现金流 (OCF)**（日常起居）、**自由现金流 (FCF)**（流动盈余）、**投资活动净现金流**（买卖资产）和**筹资活动净现金流**（偿还债务）分类的现金流入和流出明细。

### 明细与余额
- **明细列表**：按金额排序显示您最主要的收入来源和支出类别，并带有进度条指示其占总额的百分比。
- **账户余额**：列出您所有资产和负债及其当前余额的表格。点击任何账户行将带您进入其交易账簿。

## 3. 账户管理

**账户管理器** (`/accounts`) 让您可以配置财务账簿布局。Netly Ledger 支持两大类账户：

### 账户类型
1. **资产（现金、储蓄、支票）**：代表您拥有的资金。这些账户持有正余额。
2. **负债（信用卡、贷款、债务）**：代表您欠下的资金。虽然它们在报告表格中显示为负余额，但配置时使用的是正值（例如，1,200 美元的信用卡债务输入为 1,200，账簿会自动处理负数运算）。

### 创建与编辑账户
- **添加账户**：输入账户名称，选择其类型，选择货币（例如 AUD、USD、CNY），并指定初始余额。
- **初始余额**：这代表导入任何交易之前的账户余额。
- **编辑**：您可以随时修改账户名称、初始余额或货币。
- **删除**：警告 — 删除账户将永久从数据库中清除与其关联的所有交易。

## 4. 银行对账单 CSV 导入

Netly Ledger 不直接连接银行 API，以此保护您的隐私。相反，您通过**银行 CSV 导入**工具 (`/import`) 导入银行对账单文件。

### 逐步导入流程
1. **文件上传**：拖放您的银行 CSV 对账单，或点击进行浏览。
2. **配置**：
   - **标题行**：切换 CSV 是否包含标题行。
   - **目标账户**：选择这些交易所属的托管账户。
   - **日期格式**：选择与对账单文件匹配的正确布局格式（`DD/MM/YYYY`、`MM/DD/YYYY` 或 `YYYY-MM-DD`）。
   - **金额映射类型**：
     - *单列*：如果存款和取款列在同一列中，请选择此项（存款为正，取款为负）。
     - *分设贷方/借方列*：如果您的银行将存款（贷方）和取款（借方）列在不同的列中，请选择此项。
3. **列映射**：将 Netly Ledger 的标准字段映射到相应的 CSV 列：
   - **交易日期**（必填）
   - **收款人 / 描述**（必填）
   - **金额** 或 **贷方/借方列**（必填）
   - **描述列**（选填备忘录或参考信息）
4. **数据预览**：表格实时显示 CSV 的前三行解析结果，让您在提交导入之前验证日期、收款人和金额是否正确解析。
5. **自动分类**：导入时，Netly Ledger 会自动根据您保存的**匹配规则**运行收款人名称，以对交易账簿项目进行分类。
6. **重复过滤**：导入器会自动筛选交易以跳过完全重复的交易（日期、收款人、金额和账户完全匹配）。

## 5. 交易账簿

**交易账簿** (`/transactions`) 作为您的按时间顺序排列的财务日志。

### 筛选与搜索
筛选栏允许您快速隔离特定的交易子集：
- **搜索栏**：查询收款人名称、描述、类别名称或账户名称。
- **账户和类别筛选**：将结果限制在特定账户或类别。
- **仅显示未分类**：切换以仅显示等待分配类别的交易。
- **日期预设**：将视图限制为预设范围（本月、滚动 3/6/12 个月、今年以来）或选择“全部时间”。
- **审核状态**：按**仅已审核**或**仅需要审核**进行筛选。
- **货币**：按账户货币筛选交易。

### 详细交易抽屉
点击任何交易会在右侧打开**详细信息抽屉**：
- **编辑字段**：修改日期、收款人名称、备忘录描述、账户、类别和金额。
- **审核复选框**：切换“已审核”状态。未审核的项目将在应用中被标记为待审核。
- **关键字规则匹配**：如果交易匹配了某条规则，抽屉会显示它匹配了哪个模式。
- **从收款人创建规则**：直接从收款人字符串轻松生成自动分类规则。

### 批量操作
使用表格复选框选择多条交易以应用批量更改：
- **批量分类**：一次性为所有选定的交易设置类别。
- **批量删除**：永久删除所有选定的条目。
- **批量取消分类**：清除已分配的类别。

### 导入驱动的账簿架构
为了保持与现实世界银行记录的绝对一致，并防止手动输入差异错误， Netly Ledger 作为一个导入驱动的审计工具运行。您无法从头手动添加单条交易。相反，交易只能通过上传银行对账单 CSV 文件来填充。一旦导入，您可以修改它们的类别、审核状态、详细信息（例如日期、收款人关键字和金额余额）或删除它们。

## 6. 自动分类规则

为了尽量减少手动簿记， Netly Ledger 使用关键字匹配规则对传入的银行交易进行分类。

### 创建匹配规则
- **从类别管理器**（`/categories` -> **匹配规则**选项卡）：输入**商家关键字**（例如“Uber”或“Coles”）并选择目标类别。
- **从交易抽屉**：点击“从收款人创建规则”。应用将提示您保存该规则。
- **正则表达式支持**：关键字支持正则表达式。例如，模式 `^Netflix.*Subscription$` 将匹配任何以 "Netflix" 开头并以 "Subscription" 结尾的收款人。

### 配置偏好
在**设置** -> **应用偏好**中，您可以配置**自动分类模式**：
- **提示创建规则**：每当您手动分类交易时，都会弹出一个对话框模态框提示您保存规则。
- **自动创建规则**：手动分类后立即保存关键字规则，无需提示。
- **从不创建规则**：在手动输入期间禁用规则生成提示。

## 7. 重复交易对账

当银行对账单重叠时，重复的交易可能会出现在您的账簿中。Netly Ledger 包含一个强大的对账工具。

### 复合匹配
当同一账户中的多条交易具有以下相同值时，将被检测为重复交易：
- **日期**
- **收款人**
- **金额**
- **描述 / 备忘录**

### 对账工作流
如果发现重复交易，交易账簿顶部将出现一个横幅。点击**查看重复交易**将打开对账界面：
- 重复的交易会被分组在一起。
- **保留一个，删除其余**：保留单条交易并删除重复项。
- **非重复交易**：如果它们代表当天发生的合法定期费用，则驳回该分组。

## 8. 财务报表与报告

**财务报表**页面 (`/reports`) 将您的账簿记录编译为标准会计报表。

### 编译控制
- **日期选择器**：输入自定义的开始日期 and 结束日期，或选择预设范围（上个月、滚动 3/6/12 个月、今年以来、全部时间）。
- **与前一期间对比**：切换此选项可生成跨期对比，显示绝对差异和百分比差异。
- **查看货币**：由于计算必须在单一货币中进行，请选择要编译哪个货币的报表。
- **下钻模态框**：点击报表上的任何类别行会打开一个下钻模态框，显示构成该总额的所有账簿交易。

### 三大报表
1. **资产负债表**：
   - 列出资产、负债和净资产/所有者权益。
   - 留存收益自动计算为截至所选日期的历史总收入减去支出。
   - 显示您的**资产负债率**（总负债 / 总资产）。
2. **损益表**：
   - 显示收入（流入）、支出（流出）和净收入（盈余/赤字）。
   - 显示您的**储蓄率**（净收入 / 总收入）。
3. **现金流量表**：
   - 跟踪实际的现金流动，分为三个部分：
     - *经营活动*：日常现金流（生活开销、工资）。
     - *投资活动*：买卖资产（投资、设备）。
     - *筹资活动*：借贷、股权或债务偿还。
   - 对账期初现金与期末现金。

## 9. 系统设置与实用工具

**设置**页面 (`/settings`) 包含本地账簿的管理工具。

### 应用偏好
保存到浏览器的本地存储：
- **默认货币**：仪表板和报表预选的货币。
- **默认日期范围**：仪表板分析的默认期间（例如，滚动 3 个月、今年以来）。
- **首选日期格式**：设置您的日期显示格式（`DD/MM/YYYY`、`MM/DD/YYYY` 或 `YYYY-MM-DD`）。
- **界面语言**：在英文和简体中文之间切换。

### 数据库信息与维护
- **收缩并优化数据库**：重新组织磁盘上的数据库文件，回收未使用的数据库空间并优化索引布局。
- **数据库指标**：托管账户、导入交易和匹配规则的快速计数。
- **危险区域**：清空数据库 — 清除所有数据，让您重新开始。您必须输入 `WIPE` 才能执行。

### 数据导出
- **导出交易**：下载包含所有交易账簿记录的 CSV。
- **导出账户**：下载包含所有账户结构详情的 CSV。
- **导出完整账簿**：从报告页面下载包含所有数据的完整 CSV 压缩包。

### 模型上下文协议 (MCP) 访问
针对使用 AI 编码助手或本地 LLM 智能体（如 OpenCode Client）的用户：
- Netly Ledger 在 `/api/mcp` 运行一个 SSE MCP 服务器。
- **生成令牌**：在 MCP 设置卡片下，点击“生成令牌”并命名该令牌（例如“我的智能体”）。
- **配置**：在您的 AI 工具客户端配置中将此令牌用作 Bearer Auth 标头。这授予 AI 智能体安全的、程序化的访问权限，以查询您的账户、交易账簿和类别。

  例如，在 **Claude Desktop** 或 **OpenCode Client** 的配置 JSON 中：
  ```json
  {
    "mcpServers": {
      "netly-ledger": {
        "url": "http://localhost:3000/api/mcp",
        "headers": {
          "Authorization": "Bearer YOUR_MCP_TOKEN"
        }
      }
    }
  }
  ```
- **撤销**：立即撤销任何处于活动状态的令牌以阻止访问。

### 备份您的数据
由于 Netly Ledger 100% 在您的本地计算机上运行，您的数据完全由您控制。
- **数据库位置**：您的财务数据存储在相对于工作区目录的 `prisma/dev.db` 单个 SQLite 数据库文件中。
- **会话机密**：活动会话和 PassKey 验证挑战依赖于在工作区根目录中生成的 `.session-secret` 文件。
- **创建备份**：只需复制 `prisma/dev.db` 和 `.session-secret` 文件并将其存储在安全的辅助位置（例如加密驱动器或私有云存储中）。确保在复制前停止服务器容器或使其处于空闲状态，以防止部分写入。
- **恢复数据**：要恢复数据，请用备份副本替换工作区目录中的 `prisma/dev.db` 和 `.session-secret` 文件。
